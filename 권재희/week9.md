# 8. 스프링이란 무엇인가?

## 8.1 스프링의 정의
- 애플리케이션 프레임워크 : 애플리케이션 전 영역을 포괄, 개발 전 과정을 일관된 방식으로 지원
- 경량급 : 특정 컨테이너나 복잡한 인프라에 종속되지 않음, 톰캣과 같은 단순 환경에서도 동작
- 자바 엔터프라이즈 개발을 편하게 : 기술 복잡성을 제거, 생산성 향상, 이후 신경 쓸 필요 없음
- 오픈소스 : 공개 개발 모델 및 커뮤니티 기반, 빠른 진화, 유연성, 확장성

## 8.2 스프링의 목적
- 엔터프라이즈 애플리케이션 개발을 단순하고 편리하게 만드는 것
- 엔터프라이즈 개발이 복잡한 이유
    - 기술적 복잡성
    - 비지니스 로직의 복잡성
    - 두 요소가 얽혀있음
- 스프링의 복잡성 대응 전략
    - 서비스 추상화(PSA)
    - AOP
    - 객체지향 설계 지원 + DI

## 8.3 POJO 프로그래밍
- Plain Old Java Object
- POJO의 조건
    - 특정 기술·규약에 종속되지 않을 것
    - 특정 환경(Servlet/EJB/JTA 등)에 종속되지 않을 것
    - 객체지향 원리에 충실할 것
- POJO의 장점
    - 코드의 순수성
    - 테스트 용이
    - 설계 자유

## 8.4 스프링의 기술
- IOC/DI
    - 객체 생성과 의존 관계 관리를 컨테이너가 담당 -> 컴포넌트 간 결합도 최소화, 확장 및 교체 용이
    - 객체는 인터페이스에만 의존하고, 실제 구현체는 외부에서 주입 -> 테스트와 유지보수 용이
- AOP
    - 횡단 관심사 분리
    - POJO의 순수성 유지 : 비지니스 로직 코드 <-> 기술 코드
- PSA
    - 기술 종속성 제거 : 비지니스 코드 변경 최소화
    - 일관된 API 제공
